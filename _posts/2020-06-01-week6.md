Week 6: AccessHack concluded, and it was a very great experience. I attended the two panels, one themed around accessibility in academia and another themed around accessibility in industry. Being honest, I was a little shocked at the low participant numbers, especially because I thought there were some really impressive speakers who had a lot of insightful things to say. I somehow actually ended up winning first place in the competition for a new braille-based input method for non-visual text input on mobile devices. This was a fun way to start the week. 
Now returning to the data set problem, I had some conversations with Doctor Hofmann, and we settled on trying to find alternatives to AM Turk because it can be difficult to get the exact information that we needed at scale. We would need to structure tasks very specifically, and even then there wasn't a great way to ensure the quality and accuracy of the descriptions workers might write without jumping over more hoops. So instead, we first explored the option of writing a parser for openSCAD. That way, I could collect information about a 3D model if I had its corresponding openSCAD script, such as the shape, the relative/absolute positioning, etc.. Perhaps I could semi-automatically generate a data set using an openSCAD parser by processing scripts into descriptions.
